xSTUDIO Build Guides

このノートの執筆時点(2025年8月)において、xSTUDIOはソースコードのみで公開されています。そのため、アプリケーションを入手するにはソースからビルドする必要があります。

Post Production Studios and Other Organisations

多くのVFXおよびアニメーションスタジオ(特に大規模なスタジオ)には、通常、source codeからsoftwareをbuildする専門知識を持つ技術チームが存在します。softwareパッケージのbuildとdeployに関する専門知識を持つチームにとって、xSTUDIOのbuildは簡単であるはずです。以下のbuildガイドは有用なリファレンスになります。なお、xSTUDIOのバージョン1.Xは、VFX Reference Platform 2024規格に加えて、任意のプラットフォームで容易に入手またはbuildできるopen source packages/librariesの小さなセットをベースに開発されています。便宜を図るため、これらbuild依存関係のリストを提供しています。

Individual Users

技術的背景を持たない個人の方に向けて、xSTUDIOをbuildするプロセスを可能な限りシンプルにしました。以下のリストから該当するステップバイステップのガイドに従ってみてください。

ご質問がある場合は、ASWF Slackの#ori-xstudio-discussionでお問い合わせください。

Choosing a guide

macOS、Windows、およびLinux Genericのガイドはすべて同じアプローチを使用しています。xSTUDIOの依存関係はvcpkgによって自動的にbuildおよび管理されます。そのため、手動でインストールする必要があるのはコンパイラツールチェーン、CMake、git、およびQt SDKのみです。これらが推奨されるパスであり、合理的に最新のディストリビューションであればどれでも動作し、手動でのセットアップが最も少なくて済みます。

Rocky Linux 9.1、Ubuntu 22.04、およびCentOS 7のガイドは、異なる、より高度なアプローチをとっています。vcpkgを使用する代わりに、各ディストリビューション固有のパッケージマネージャーからxSTUDIOの依存関係を解決します(また、適切なパッケージが存在しない場合は、いくつかのライブラリをソースからbuildします)。これにより、ホストシステムとのより緊密な統合が可能になりますが、作業は大幅に増え、脆弱になります。パッケージ名やバージョンは時間の経過とともに変化し、VFX Reference Platformとの不一致が発生した場合は手動での対応が必要になるためです。これらのガイドは、特にネイティブパッケージでのbuildを希望するユーザーや、ターゲットとするディストリビューションがこれら3つのいずれかに一致し、パッケージの問題のトラブルシューティングに慣れているユーザーを対象としています。Linuxマシン上で単にxSTUDIOをbuildしたいだけの場合は、Linux Genericを優先してください。
