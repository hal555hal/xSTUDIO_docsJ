Colour Management

xSTUDIOは、OpenColorIO (OCIO)フレームワークとの統合により、colour management機能を備えています。適切な構成ファイルを配置することで、ディスプレイデバイス(モニター、プロジェクターなど)のカラープロファイルとメディアソースのカラースペースが正しく結合され、最終的な画像が正確に表現されます。ピクセルが画面に表示される前に、グレーディング値やフィルムエミュレーションLUTなどのアーティスティックなルックの変更を組み込むこともできます。xSTUDIOは、OCIOの最新のV2機能を活用して、グラフィックスカード上でカラースペースの変換を計算し、LUTを適用するため、確実な正確性と高いパフォーマンスが保証されます。

Note

xSTUDIOにはいくつかのデフォルトのOCIO構成が同梱されており、これには標準の`OpenColorIO ACES <https://en.wikipedia.org/wiki/Academy_Color_Encoding_System> config.`_が含まれています。環境設定マネージャー(Preferences Manager)のColourタブから、どの構成をアクティブにするかを選択できます。

Note

高度なcolour managementが必要ない場合：xSTUDIOはOCIO機能を有効にしなくても、out-of-the-boxで動作します。インターフェース上部のメインメニューバーにあるColourメニュー項目から、「Bypass Colour Management」オプションをチェックしてください。

以下のガイドは、有効なOCIO構成が配置されている場合に適用されます：

  • ツールバーの「Display」オプション：このツールバーボタンを使用して、リストからご使用のディスプレイハードウェアに一致する適切なディスプレイデバイスを選択します。選択内容はローカルの環境設定ファイルに保存され、セッション間で保持されます。

  • ツールバーの「View」オプション - このツールバーボタンを使用して、組み合わされたViewとLookを選択します。

colour managementシステムは、表示されているメディアのカラースペースをご使用のディスプレイデバイス(ノートPCの画面/PCモニターなど)のディスプレイカラースペースに変換することによって動作します。そのため、OCIOはメディアのカラースペースを認識する必要があります。これを自動的に推測するいくつかのロジックが備わっていますが、ソースカラースペースを手動で設定することもできます。これは、Viewport内でマウスを右クリックする(またはViewportの右上にあるMoreボタンをクリックする)か、OCIO->Sourceサブメニューにアクセスすることで実行できます。

OCIO Configuration (For Developers)

これらの機能を活用するには、必要なOCIO構成ファイルと、それに関連付けられたLUTデータファイルがfilesystem上に存在する必要があります。ほとんどのVFXスタジオやその他のポストプロダクション施設では、OCIOデータの構成は専門の個人またはチームによって管理されており、そのような技術的専門知識がすぐに利用できることを前提としています。詳細については、ocioのウェブサイトを参照してください。

Note

xSTUDIOへのOCIO統合はpluginとして実装されています。これは、独自のOCIO実装やその他のcolour managementシステムを追加したいスタジオが、それを行えるようにするために実施されました。xSTUDIOに同梱されているOCIO pluginコードは、この場合のリファレンス実装として機能します。

慣例に従い、xSTUDIOは$OCIO環境変数に格納されたパスから構成ファイルをロードします。python APIを使用するか、必要なメタデータを追加するための「media hook」C++ pluginを実装して、メディアに特別なメタデータ値を設定することにより、これをメディアソースごとにオーバーライドできます。本質的に、「media hook」pluginは、xSTUDIOが新しいメディアをロードするときはいつでも実行されるコールバック関数を提供します。この関数は、新しいメディアアイテムのメタデータにマージされるjsonディクショナリを返します。これは、作成時にメディアにカスタムメタデータを追加することを可能にするメカニズムの1つ(いくつかあるうちの1つ)です。

このカスタムメタデータの一部として、OCIO pluginが検索する文字列のキー/値のペアを含む「colour_pipeline」ディクショナリを追加できます。xSTUDIOのOCIO pluginは、これらのキー/値のペアを使用して、OCIOプロセッサ内のcontextオブジェクトを初期化するだけでなく、どのOCIO構成ファイルをロードして適用するかを決定します。たとえば、SHOTキーを使用してグレーディングデータを見つけるために、ファイルパス一致ルールを使用するようにOCIO構成がセットアップされている場合、media hook pluginでSHOTの値を設定します。参照用のDNEG独自の実装については、xSTUDIOソースコードツリー内の「dneg.cpp」ファイルを参照してください。

メディアメタデータに「ocio_config」値を設定して、OCIO環境変数をバイパスし、代わりにソースごとにOCIO構成を設定することもできます。

Example showing how to set OCIO config and context overrides on all media sources in a session
from xstudio.connection import Connection

XSTUDIO = Connection(auto_connect=True)
playlists = XSTUDIO.api.session.playlists

colour_management_metadata = {
    'input_display': 'Rec709',
    'input_view': 'Film',
    'ocio_config': '/home/colour_data/config.ocio',
    'ocio_context': {
        'SHOT': 'shot_001',
        }
    }

for playlist in playlists:

    media = playlist.media
    for m in media:
        metadata = m.media_source().metadata
        metadata['colour_pipeline'] = colour_management_metadata
        m.media_source().metadata = metadata
