xSTUDIO Media Item Structure

VFXやアニメーションのpipelinesでは、同じレンダリング、スキャン、プレイブラストなどの複数のエンコーディングがディスク上に存在することが一般的です。たとえば、CGレンダリングは、レンダラーやコンポジットアプリケーションによって出力された高解像度・高ビット深度のEXRイメージのセットとしてファイルシステム上に存在する場合があります。また、xSTUDIOのようなプレイヤーへのストリーミングがはるかに容易な、QuickTimeムービーとしてこれらのEXRイメージをポストレンダリングで生成した、より軽量な圧縮エンコーディングが存在する場合もあります。あるいは、解像度が半分または4分の1の同一のEXRシーケンス(通常はproxiesと呼ばれます)が存在することもあります。特定のクライアントへの納品物や社内pipelineの要件をサポートするために、同じ出力に対して多数の並列エンコーディングが存在する場合もあります。これは完全にスタジオ独自のpipelineに依存します。

画像をレビューする際、xSTUDIOのユーザーは、レビュー活動に応じてこれらのエンコーディングのいずれかを表示したいと考える場合があります(animation dailies reviewでは軽量なエンコーディングのみが必要な場合があり、lighting dailiesではすべてのピクセルが必要になる場合があります)。

xSTUDIOのコアデータモデルは、VFXおよびアニメーションスタジオにおけるこの一般的な慣行を反映するように設計されています。そのため、xSTUDIOにおけるMedia Item(プレイリストに個別の独立したアイテムとして表示されるオブジェクトを意味します)は、内部的に1つまたは複数のMedia Sourcesで構成できます。xSTUDIOのインターフェースおよびAPIは、Media Itemに対してどのタイプのMedia Sourceをアクティブにするかを迅速に切り替える機能(ビューポートツールバーの「Source」ボタンなど)を提供します。

xSTUDIOユーザーがこの機能を活用できるようにするには、特定の出力バージョンのさまざまなエンコーディングを検索する方法を認識し、ユーザーがそれをロードしたいときにそれに応じてMedia Itemをbuildする、いくつかのpipeline統合pluginsをxSTUDIOに拡張する必要があります。PythonまたはC++ APIを介して、複数のMedia Sourcesを持つメディアをbuildして追加することが可能です。

確立されたpipelineや関連するインフラストラクチャの外で作業するxSTUDIOユーザーの場合、単一のMedia SourceをラップするMedia itemsがシームレスに処理されるため、xSTUDIOのこの機能は無視して構いません。
