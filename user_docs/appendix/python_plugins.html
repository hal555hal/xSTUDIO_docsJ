Python Plugins

Introduction

xSTUDIOにはPython plugin APIがあり、独自のxSTUDIO拡張をbuildするために使用できるpluginベースクラスが提供されています。pluginベースクラスを使用すると、アプリケーションやセッションデータと相互作用するための多くの方法が利用可能になります：

  • xSTUDIOのcore session obejctとの相互作用

    • Playlistsのbuildおよび管理

    • 高速なロードのための並行メディア生成

    • マルチトラックタイムライン用のSequenceオブジェクトのbuildまたはロード

    • タイムライン交換用のOpenTimelineIOファイルのインポート/エクスポート

  • xSTUDIOの再生エンジン、viewport、およびcolour managementコンポーネントとの相互作用

    • 再生の開始/停止、再生ヘッドの配置、メディアのスキップ

    • メディア比較(A/B、Wipe、Grid(Contact Sheet))モードの動的なセットアップ

    • colour managementデータの動的な制御(露出の変更、gradesの適用)

    • メインインターフェースの外で個々のメディアアイテムをレビューするための「QuickView」プレイヤーウィンドウの生成

  • pluginへのattribute data項目の追加
    • Attributesは、幅広いデータ型を表すことができる柔軟な複合データ型です：

      • boolean

      • integer number

      • string

      • real number

      • list

      • dictionary(Jsonフォーマット可能)

    • attribute dataはQMLコードから直接アクセス可能

    • AttributesのセットはQML内で、QtのModel-View-Delegateデザインパターンに沿った動的な「model」データとしてアクセス可能

  • Qt QMLを介して、xSTUDIOのUIに埋め込まれるカスタムUI「panels」を作成

  • カスタムのQt QMLフローティングウィンドウまたはダイアログを作成

  • 必要なあらゆるデータの可視化のために、Viewport用のカスタムオーバーレイQMLグラフィックスを作成

  • xSTUDIOのメニューシステムにメニュー項目をインジェクト

  • トリガー時に独自のコールバックを実行するhotkeyショートカットを追加

  • UIコード(Qt QML)からコールバックを受信

  • Viewportツールバーにカスタム項目を追加

  • 各種xSTUDIOコンポーネントからブロードキャストされるイベントメッセージのサブスクライブ

    • 画面上のMediaアイテムが変更されたときに通知を受信

    • 再生ヘッドの状態(位置、再生中、比較モードなど)が変更されたときに通知を受信

    • 新しいViewportが作成されたときに通知を受信

Note

xSTUDIOの組み込みPython interpreterはメインのUIイベントループとは別のスレッドで実行されるため、Pythonコードの実行によってxSTUDIOのインターフェースが遅くなったりフリーズしたりすることはありません。Python plugin内で大量の新しいメディアをロードする大きなplaylistsをbuildするコードを実行している最中でも、xSTUDIOのViewportやplayback engineが遅延することはないため、フォアグラウンドでスムーズかつ正確な再生を継続できます。

Authoring Plugins

xSTUDIOは起動時にPython pluginsをロードします。環境設定マネージャー(Preferences Manager)のPythonセクションにある環境設定(Python Plugin Path)で、xSTUDIOがpluginsをスキャンするための1つ以上のファイルシステムディレクトリを設定できます。これに加えて、xSTUDIOは環境変数XSTUDIO_PYTHON_PLUGIN_PATH(Mac/Linuxではコロン区切り、Windowsではセミコロン区切りのリスト)のディレクトリも検索します。

各pluginは、独自のフォルダ内にPythonのインポート可能なモジュールとしてインストールする必要があります。pluginのPythonモジュールのインターフェースには、pluginクラスのインスタンスを返すメソッドcreate_plugin_instanceを含める必要があります(以下の例を参照してください)。

Bare Bones Plugin Example

たとえば、環境設定のPython Plugin Pathにディレクトリ/home/Users/tedwaine/xstudio_pluginsが追加されているとします。このディレクトリ内に、my_pluginという名前の別のディレクトリがあるとします。さらにその中に、以下のようなdemo_plugin.pyという名前のファイルを作成できます：

demo_plugin.py - minimal plugin that adds a menu option to the main menu bar
from xstudio.plugin import PluginBase
from xstudio.core import KeyboardModifier

# Declare our plugin class - we're using the PluginBase base class here which
# is a general purpose plugin
class MyDemoPlugin(PluginBase):

    def __init__(self, connection):

        PluginBase.__init__(
            self,
            connection,
            "My Demo Plugin")

        self.shortcut_id = self.register_hotkey(
            self.shortcut_callback,
            "W",
            KeyboardModifier.ControlModifier,
            "Demo Plugin Shorcut",
            "Simple example hotkey",
            auto_repeat=False,
            component="Demo Plugins",
            context=None)

        self.insert_menu_item(
            "main menu bar",
            "Run Callback",
            "Plugins|Demo Plugin",
            0.0,
            hotkey_uuid=self.shortcut_id,
            callback=self.my_callback,
            user_data="some extra info")

        # ensure that the new 'Plugins' section of the menu bar appears
        # at the end (right side) of the menu bar
        self.set_submenu_position(
            "main menu bar",
            "Plugins",
            1000.0)

    def shortcut_callback(self, shortcut_uuid, context):

        print ("Hotkey pressed in context {}".format(context))

    def my_callback(self):

        print ("my_callback executed")

    def menu_item_activated(self, menu_item_data, user_data):

        print ("my_callback executed with user data", menu_item_data, user_data)

# This method is required by xSTUDIO
def create_plugin_instance(connection):
    return MyDemoPlugin(connection)

filesystemの同じ場所に、xSTUDIOのpluginローダーに対して重要なcreate_plugin_instanceを公開するための__init__.pyファイルも必要です：

__init__.py -
from .demo_plugin import create_plugin_instance

Note

上記の例のようなpython pluginを開発する場合、XSTUDIO_PYTHON_PLUGIN_PATHを設定して、pluginのコードが含まれるフォルダの1つ上の階層にある開発フォルダを指すようにすることができます。コードを変更した後は、xSTUDIOを再起動するだけでテストできます。pluginのインポート時に構文エラーやその他のランタイムの問題が発生した場合、エラーはターミナルにプリントされます。このため、開発サイクルを迅速かつ容易にするには、コマンドラインでxSTUDIOを実行できる必要があります。いくつかのヒントがGetting Started sectionに記載されています。
